# QA-Ecommerce-Testing
End-to-end QA testing project for an e-commerce application covering test planning, test case design, and API testing with focus on real-world edge cases and risk-based validation.

## Overview
This project demonstrates a structured approach to testing an e-commerce web application, covering the complete QA lifecycle from requirement analysis to test design and API validation.

The goal is to simulate real-world QA practices by focusing on high-risk areas such as cart calculations, checkout flow, and payment processing.

---

## Scope of Testing
The following modules are covered:

- User Authentication (Login, Registration)
- Product Search and Catalogue
- Cart Management
- Checkout Flow
- Payment Processing
- Order Handling

---

## Key Work Completed

### 1. Requirement Analysis
- Created a BRD-lite document to define system behavior and scope

### 2. Test Planning
- Developed a detailed Test Plan including scope, strategy, risks, and coverage

### 3. Test Case Design
- Designed 30+ structured test cases
- Covered:
  - Positive scenarios
  - Negative scenarios
  - Edge cases (session timeout, duplicate order, payment failure)

### 4. API Testing
- Built Postman collection for:
  - Product APIs
  - Cart APIs
  - User APIs
- Added validations for:
  - Status codes
  - Response structure
  - Negative scenarios

---

## Key Highlights
- Focus on **high-risk areas** like payment and checkout
- Included **real-world edge cases** (network failure, duplicate transactions)
